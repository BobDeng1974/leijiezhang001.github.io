如何搭建一个 ADAS 产品

置顶 | Posted on 2020-05-07   |   Updated on 2020-05-12 | In Autonomous Driving System | 本文总阅读量 次
Words count in article: 2.8k 字 | Reading time ≈ 9 分钟

　　ADAS(Advanced Driver Assistance System) 高级辅助驾驶系统是指 L3 及 L3 以下的汽车自动化系统。前装 ADAS 一般也包括感知，决策，控制三个部分，而后装 ADAS 则只包括感知，决策两部分，且各功能的决策出口一般就是声音或视觉警报。本文讨论后装 ADAS 系统的设计与搭建过程。
　　一个 ADAS 产品的搭建一般涉及到硬件及结构，平台系统，算法功能等。市面上比较有名的 ADAS 产品有基于单目的 Mobileye，Minieye，极目智能等，基于双目的中科慧眼，双髻鲨等，本文讨论一种三目结构的 ADAS 产品搭建过程。

1. 硬件及结构

　　单目 ADAS 与双目 ADAS 各有优劣，目前最成功的 Mobileye 是基于单目的产品，其优势是成本低，探测范围广，缺点是标定麻烦，测距精度低，算法复杂；而双目的优势是容易自动化标定，测距精度高，缺点则是成本高，探测范围受基线限制。

1.1. 镜头

1.1.1. 参数

　　镜头的作用是汇聚光线到 Sensor 成像。镜头的主要参数有：

  • 焦距，焦距决定成像大小，视场角大小，景深大小和画面的透视强弱。
  • FOV(视场角)，分为水平视场角 HFOV 和垂直视场角 VFOV，一般关注 HFOV，焦距越大，HFOV 越小。
  • F 值(口径比)，F = 镜头焦距/光圈直径，表示镜头的透光量。在相同透光量下，长焦的口径要比短焦大。
  • 光圈，用来调节进光量。
  • 景深，是指当某一物体聚焦清晰时，在该物体前后一段距离内所物体都是清晰的，这段距离叫做景深。

1.1.2. 指标

镜头指标有：

  • 耀斑，鬼影，炫光。镜头反射，色散会导致一些列的耀斑，鬼影，炫光的情况，尤其在逆光的场景中。这可以通过镜头表面镀特定的膜，加偏正片等方法来缓解或解决。
  • 画质清晰度。通常用 MTF 曲线来评价，最终画质的清晰度是整个成像过程所决定的，比如还要高像素，每个环节都可通过 MTF 评估。
  • 感光区要大。经过镜头的光要覆盖 Sensor 感光范围。
  • CRA(Chief Ray of Angle，主光线角度)。镜头与 Sensor 的 CRA 要尽可能一致，以减少 Shading 现象。Shading 是指成像中心比周围更加聚光而导致的光强及色彩偏差。
  • 光圈。光圈尽可能大，以透过更多的光。
  • 畸变。广角镜头容易产生桶形畸变，长焦镜头容易产生枕形畸变。畸变要尽可能小，当然，一般相机内参标定都能校正畸变，但是导致图像边缘的裁剪。

1.1.3. 景深细节

　　如图 1. 所示，在焦点前后，光线开始聚集和扩散，点的影像变得模糊，形成一个扩大的圆，人眼能够辨认最小的圆称为容许弥散圆，蔡司公司制定的标准是，弥散圆直径\(=\frac{1}{1730}\times\)底片对角线长度。焦距越大，光圈越大，景深越浅。
　　景深与对焦点的关系则比较复杂(对焦是指调节成像面与透镜距离的过程)，首先引入超焦距概念。如图 2. 所示，当镜头对焦在无穷远时，景深前界(离镜头最近清晰点)到镜头的距离称为超焦距。当对焦到超焦距点时，获得的景深更深，即近处的盲区更小。超焦距的计算公式为： \[超焦距 = 焦距 + \frac{焦距 ^2}{弥散圆直径\times 光圈} \tag{1}\] 实际定焦镜头可以直接推算光圈、对焦点、景深的关系。 如图 3. 所示，当光圈为 8 时，对焦在无穷远处，则前景深为 \((3, \infty)\)，无后景深，由定义知此时 3 即为超焦距；当对焦到超焦距 3m 时，前景深为 \((1.5,3)\)，后景深为 \((3,\infty)\)，总的景深为 \((1.5, \infty)\)。所以对焦到超焦距，能获得更小的近处盲区。

1.2. Sensor

　　ADAS 或自动驾驶应用场景下要求 Sensor 有高感光度(High Sensitivity)，高动态范围(High Dynamic Range)。Sony 官网已经有几款适合自动驾驶场景的 Sensor 可供选择。高感光度及高动态范围也可以通过后期 ISP 改善，但是这样会引入较大噪声，所以如果 Sensor 本身低光照下感光度较高，出入隧道时动态范围较宽，那么后期获得的图像质量就会很高。
　　此外 Sony 官网[1] 中还提到了频闪消除(LED Flicker Mitigation)，理论上这个可以通过控制曝光时间消除，下面 ISP 章节再做介绍。

1.3. 结构 ID

　　结构 ID 需要考虑的方面有：散热，美观，遮光。其中遮光在车内后装产品中非常重要，因为挡风玻璃会将车内的物体反射到镜头内，所以我们看到市面上的 ADAS 产品基本都是都有很长的遮光罩，安装时紧贴挡风玻璃。但是这样也会降低产品的美观成都，所以要不要遮光罩也得看是否影响算法功能，比如对基于深度学习的检测算法基本没有影响，但是对于基于双目的立体匹配算法，影响较大。此外，还有其它方式可以消除或减轻该影响，一是镜头加入偏振片，二是算法端作背景滤出处理，因为时间序列上反射的物体模式是固定的。

1.4. 三目硬件选型与设计

　　三目 ADAS 系统由 RGB 主摄及灰度双摄构成，主摄主要用于行车记录仪功能，双目用于目标测距及相关功能实现。各硬件选型的准则为：

  • 镜头材质。玻璃材质耐高温成本高，塑胶材质不耐高温成本低，由于车内容易出现高温，所以一般选择玻璃材质。
  • 对焦。因为需要远近均能清晰成像，所以要求景深要深，为获得最大的景深，选择对焦到超焦距处，进一步权衡光圈与近处盲区，在焦距确定的情况下，增大光圈，直到盲区符合指标要求。
  • 焦距/视野(FOV)。主摄因为有行车记录仪的需求，所以可选择 HFOV 大于 100° 的镜头，覆盖 A 柱盲区；双目的 HFOV 可以根据功能需求选择，主要有两方面决定，看远要求 FOV 小，看侧面切入的车则要求 FOV 大，从中作权衡。
  • Sensor。在成本允许的情况下，要选高感光度高动态范围的型号。主摄选 4K 像素的，这样在大 FOV 下远处分辨率依然较高。
  • 快门。主摄的快门可以选用便宜的 rolling shutter，而双目对像素级别的曝光一致性要求较高，所以必须选用 global shutter。
  • 结构。结构首先要满足功能需求，当然如果算法功能能克服挡风玻璃反射的问题，那么去掉遮光罩会极大提升 ID 美观度。

2.1. 3A+ISP

　　3A 指 AE，AF，AWB，自动驾驶应用场景中，不需要自动变焦(上文提到是定焦的)，自动白平衡也是与传统需求一致的，这里比较棘手的是自动曝光 AE 的调节。由于光圈与焦距，以及快门时间是固定的，所以 AE 只可通过 ISO，增益放大系数调节。AE 本质上是一个负反馈系统，可通过 PID 算法进行控制调节，对 AE 的要求是“快，准，稳”。即控制系统的曲线响应要快，能将被控变量-曝光量调节到准确的值，最后曲线震荡收敛要稳。

3. 算法功能

　　ADAS 功能模块主要有车道偏离预警系统(LDWS，Lane Departure Warning System)，前碰撞预警系统(FCW，Forward Collision Warnings)，自适应巡航(ACC，Adaptivecruise Control)等。这里主要介绍基于三目的 LDWS，FCW 功能，以及红绿灯提醒/前车启动提醒小功能。

3.0. 标定

　　三目系统的内外参离线标定已经比较成熟，这里不作展开。其中双目系统中立体视觉对外参精度要求比较高，所以如果双目之间没有刚体连接(DJI 无人机的双目系统目前基本是刚体连接的，所以应该没做在线自标定)，那么外参的在线自标定就非常重要。
　　一般的方法是左右目提取特征点，然后最小化重投影误差，从而以较低频率来更新双目之间的外参。

3.1. LDWS

3.1.1. 需求描述

　　借鉴华为的产品功能需求分析方法，原始需求->初始需求->系统特性->系统需求。原始需求是产品经理对产品的功能定义及需求；初始需求是工程师针对原始需求分解的算法功能需求；系统特性是工程师对算法功能设定的指标特性；最后，系统需求是工程师为了满足系统特性而分解的最终系统需求。
　　LDWS 功能的目的：通过对车道偏离情况的提前预警，降低用户在无意识下偏离车道带来的危险。其原始需求有：

  • 仅在一定速度以上时开启该功能(如 55km/h);
  • 对报警实际，精度要求，车道线类别及弯道曲率要求，应符合 JT/T 883, GT/T 26773-2011，ISO 17361 标准；
  • 夜晚正常工作；
  • 暴雨、沙尘，大雪等天气，可允许漏报，但是不能误报；
  • 雨刷、弯道、掉头时，不会导致误报；
  • 当可以获取到车辆转向灯时，转向灯开启时忽略车道线偏离预警；

　　为了满足原始需求，可列一些必要的初始需求：

  • 车道线识别；
  • 报警交互；
  • 系统监控；
  • 感知融合，标定等反馈模块；
  • 测试功能；
  • 性能要求符合特定指标；
  • 生产相关：如生产过程中的初始化标定流程；

　　由此可进一步推演出系统特性与系统需求，这里不作展开。

3.2. FCW

3.3. 红绿灯提醒 & 前车启动提醒

　　前车启动提醒的基本算法模块与 FCW 一致，主要是注意近处盲区的问题。红绿灯提醒包括红绿灯检测与决策提醒两部分。
